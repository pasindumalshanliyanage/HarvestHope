Feedback Page

Technical Description

The Feedback Page is designed to collect user feedback through a structured form. The HTML structure uses <form> to encapsulate the feedback fields, with <fieldset> and <legend> to group related inputs logically. Key HTML tags include:

  • <input>: Used for text, email, and telephone inputs, with type attributes specifying the input type.
  • <textarea>: For longer text inputs like comments and additional feedback.
  • <select>: For dropdown menus, such as feedback type and update preferences.
  • <label>: Associates text labels with form elements for better accessibility.
  • <div> and <span>: Used for structuring and styling elements like the star rating system.
CSS is used to style the form for a cohesive and user-friendly design. Key CSS properties include:
  • grid and flexbox: For layout management, ensuring the form is responsive and well-organized.
  • box-shadow and border-radius: To create a modern, visually appealing form with rounded corners and subtle shadows.
  • transition: For smooth hover effects on buttons and interactive elements.
  • display: none and display: block: Used to show/hide the update preferences section based on user selection.
The JavaScript (js.js) handles form validation and interactivity, such as:
  • Email validation: Ensures the email input follows a valid format.
  • Star rating system: Allows users to click stars to rate their experience.
  • Dynamic form behavior: Shows/hides the update preferences section based on user input.

Accessibility

Accessibility was addressed by:

  • Labels and Inputs: Each form element is paired with a <label> using the for attribute, ensuring screen readers can associate labels with inputs.
  • ARIA Roles: Although not explicitly added, the form structure inherently supports accessibility by using semantic HTML.
  • Focus States: Interactive elements like buttons and inputs have visible focus states, aiding keyboard navigation.
  • Error Messages: Clear error messages are displayed for invalid inputs, helping users correct mistakes.

Team Page

Technical Description

The Team Page showcases team members in a grid layout. The HTML structure uses <div> elements to create a grid of team member cards. Key HTML tags include:

  • <img>: Displays team member photos.
  • <h3> and <p>: For member names, roles, and task details.
  • <div>: Used to structure the grid and individual cards.
CSS is used to create a visually appealing and interactive layout. Key CSS properties include:
  • grid-template-columns: Defines a 4-column grid for team members, ensuring a responsive layout.
  • transform: scale(1.05): Adds a hover effect to enlarge team member cards.
  • position: absolute and transform: translateX(-50%): Positions member info at the bottom of each card.
  • box-shadow and border-radius: Enhances the visual appeal of the cards.
  • transition: Smoothly animates hover effects.

Accessibility

Accessibility considerations included:

  • Alt Text: Each <img> includes alt text, describing the team member for screen readers.
  • Focus States: Team member cards are focusable (tabindex="0"), allowing keyboard users to interact with them.
  • Semantic HTML: Proper use of headings (<h2>, <h3>) ensures a clear content hierarchy for screen readers.
  • Contrast: Text and background colors are chosen to ensure sufficient contrast for readability.

Content Page

Technical Description

The Content Page (content_ST3.html) presents information about the impact of hunger on communities. The HTML structure uses <section> to divide content into logical sections, with <h2> for section headings and <p> for paragraphs. Key HTML tags include:

  • <img>: Displays relevant images with alt text for accessibility.
  • <a>: Used for internal navigation links within the page.
  • <div>: Wraps content sections for styling and layout.
CSS is used to style the page for readability and visual appeal. Key CSS properties include:
  • max-width and margin: auto: Centers the content container on the page.
  • box-shadow and border-radius: Adds depth and rounded corners to sections and images.
  • grid-template-columns: Used in the internal navigation to create a 2-column layout for links.
  • position: fixed: Positions the "Go to Top" button at the bottom-right corner of the page.

Accessibility

Accessibility was addressed by:

  • Internal Navigation: The internal navigation links (<a>) allow users to jump to specific sections, improving navigation for keyboard and screen reader users.
  • Alt Text: All images include descriptive alt text.
  • Contrast: Text and background colors are chosen to ensure readability.
  • Focus States: Interactive elements like links and buttons have visible focus states.

Challenges and Lessons Learned

During the implementation of the Feedback, Team, and Content pages, I encountered several challenges:

  • Form Validation: Implementing real-time email validation required careful handling of input events and error messages.
  • Responsive Design: Ensuring the team page grid and feedback form were responsive across devices required extensive testing.
  • Accessibility: Balancing visual design with accessibility requirements, such as contrast and focus states, was challenging.
These challenges taught me the importance of planning and testing early in the development process.

Compliance

The pages I developed comply with Janet regulations by:

  • Accessibility: Ensuring all pages are navigable via keyboard and screen reader-friendly.
  • Validation: Using valid HTML and CSS, as confirmed by W3C validation tools.
  • Content Clarity: Providing clear and concise content with proper headings and labels.
  • Security: Ensuring forms are secure and user data is handled responsibly (though no backend is implemented, this would be a consideration in a live environment).
